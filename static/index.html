        How each score is calculated
        

          
Latency

          

            Wall-clock time for the full pipeline: input validation → Cypher generation
            (GPT-4o) → Neo4j query → insight synthesis.
          


          
Row Count

          

            Number of rows returned by the generated Cypher query (capped at 100 by
            the database layer).
          


          
Relevance

          

            GPT-4o-mini judge — prompt: "Does the answer directly address the question?"

            Verdict → score: YES → 1.0   PARTIAL → 0.5   NO → 0.0
          


          
Creativity

          

            GPT-4o-mini judge — prompt: "Does the answer surface non-obvious insights beyond a plain list?"

            Verdict → score: HIGH → 1.0   MEDIUM → 0.5   LOW → 0.0
          


          
Latency Score

          

            Deterministic formula: max(0, 1 − latency / 10) — target ≤ 10 s, score
            hits 0 at 10 s.

            Example: 6.7 s → 1 − 6.7/10 = 0.330
          


          
Composite

          

            Equal-weighted average of the three scored dimensions:

            (Relevance + Creativity + Latency Score) / 3

            Example: (1.00 + 1.00 + 0.330) / 3 = 0.777

            Accuracy is omitted here — no ground-truth assertion is
            available for ad-hoc questions. The full eval suite (eval/evaluate.py) adds
            Accuracy (30 %) via result_assertion callables on 20 labelled test cases.
